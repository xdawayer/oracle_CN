## Context

星座配对页面是 AstroMind 应用"发现"模块的核心功能之一，用于分析两个人的星座和属相配对契合度。当前实现存在 UI 风格不统一、爱情主题不突出、信息展示不完整等问题。

### 约束条件
- 必须遵循 `COLOR_SYSTEM_GUIDE.md` 中的设计规范
- 微信小程序环境限制
- 星座图标必须使用 PNG 图片（参见 CLAUDE.md 技术规范）

### 参考页面
- `miniprogram/pages/daily/daily.wxml` - header 布局参考
- `miniprogram/pages/self/self.wxml` - 页面结构参考

## Goals / Non-Goals

### Goals
- 统一页面 header 布局风格
- 强化爱情主题视觉表达
- 完善星座+属相的信息展示
- 提升用户体验和情感共鸣

### Non-Goals
- 不改变核心配对算法逻辑
- 不增加新的页面路由
- 不修改数据存储结构

## Decisions

### Decision 1: Header 布局采用 daily/self 页面风格
**选择**：使用简洁的居中标题 + 副标题结构

**理由**：
- 与应用其他页面保持一致
- 减少视觉噪音，突出核心功能
- 移除不必要的返回按钮（可通过系统导航返回）

**替代方案**：
- 保留原有三栏布局 - 放弃，因为与其他页面不一致

### Decision 2: 中央装饰使用爱心符号
**选择**：使用 ❤️ 符号配合心跳动画

**理由**：
- 直观表达爱情配对主题
- 增强情感共鸣
- 符合用户对配对功能的心理预期

**替代方案**：
- 使用 VS 文字 - 放弃，缺乏情感表达
- 使用星座连线图 - 放弃，过于复杂

### Decision 3: 星座与属相显示比例 3:1
**选择**：星座名称 48rpx，属相 16rpx

**理由**：
- 星座是主要信息，应突出显示
- 属相作为补充信息，辅助展示
- 3:1 比例在视觉上平衡且层次分明

### Decision 4: 爱情主题色彩方案
**选择**：基于现有设计系统，使用 `var(--danger)` 作为主色调

**理由**：
- 复用现有颜色变量，保持一致性
- 红/玫瑰色系符合爱情主题
- 避免引入过多新颜色变量

## Risks / Trade-offs

### Risk 1: 风格变化可能影响用户习惯
**缓解**：保持核心交互流程不变，仅优化视觉呈现

### Risk 2: 爱心动画可能影响性能
**缓解**：使用 CSS 动画而非 JS 动画，控制动画复杂度

### Trade-off: 简化 vs 信息完整
选择在卡片中同时显示星座和属相，虽然增加了视觉复杂度，但提供了更完整的信息展示。

## Migration Plan

1. 备份现有代码
2. 分阶段实施 UI 变更
3. 每个阶段完成后进行视觉验证
4. 保持向后兼容，不改变数据结构
