# Change: 丰富人生K线报告解读内容

## Why

当前人生K线模块中，"命运山河"点击年份节点弹出的报告内容过于简略（尤其是四维分析每条只有1句话），"人生长卷"六章报告的文字量也偏少。用户反馈解读不够详细，无法获得足够的参考价值。需要在保持通俗易懂的前提下，大幅丰富解读内容。

## What Changes

### 1. 命运山河 - 年度详情弹窗（前端 `kline.js`）

- **四维分析文案扩充**：每个维度（事业/财运/感情/健康）的分析从当前的1句话扩展为4-5句话，包含：运势概述、具体表现、时间节奏（上/下半年差异）、实用建议
- **占星视角扩充**：从1句话扩展为3-4句话，增加宫位能量解读和具体建议
- **八字视角扩充**：从1句话扩展为3-4句话，增加五行互动解读和实用指引
- **个性化寄语扩充**：增加更多分数区间的差异化寄语，使内容更贴合用户实际状况
- **行动建议优化**：每条建议从短语扩展为带简短说明的完整句子

### 2. 人生长卷 - 六章报告（前端 `kline.js`）

- **各章节文字量翻倍**：将 `generateReportContent` 中六个章节的内容从当前约200-350字/章扩展到约400-700字/章
- **overview（人生运势总览）**：增加三大星体的详细解读、元素能量分析
- **past（往昔回溯）**：每个人生阶段增加2-3句详细描述，补充具体生活场景
- **present（当下定位）**：增加更多维度的机遇与挑战分析
- **future（未来三十载）**：每个时期增加具体领域预测和行动指引
- **milestone（人生里程碑）**：每个节点增加"如何应对"的实用建议
- **letter（予未来之我）**：增加更多与本命盘相关的个性化内容

### 3. 后端文案同步（`backend/src/services/kline.ts`）

- 后端 `generateMajorEvent`、`generateActionAdvice`、`generateAstroSummary`、`generateBaziSummary`、`generatePersonalMessage` 等函数同步更新，保持前后端一致

## 内容风格要求

- **避免专业术语**：将占星/命理术语翻译为日常用语（如"土星回归"解释为"人生结构调整期"）
- **接地气的表达**：使用年轻人熟悉的语境和比喻
- **心理学导向**：侧重自我认知和成长，避免宿命论表述
- **实用性优先**：每段解读都包含可操作的建议

## Impact

- 受影响的文件：
  - `miniprogram/pages/kline/kline.js`（主要改动 - `generateYearlyReport`、`getDimensionAnalysis`、`generateReportContent`）
  - `backend/src/services/kline.ts`（同步改动 - 各文案生成函数）
- 不涉及 API 接口变更
- 不涉及数据结构变更
- 不涉及样式变更
- 纯文案层面的内容丰富
